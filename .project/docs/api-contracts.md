# API & Sync Contracts

> **Phase 1 note:** These contracts are defined now to avoid design debt later, but they are **not implemented** until the Core Module (backend) is built. In Phase 1, the Electron kiosk seeds its own SQLite data locally.

All payloads must be strictly typed:
- **Laravel side:** `Spatie\LaravelData\Data` classes
- **Electron side:** TypeScript interfaces in `src/shared/types/`
- All monetary values are **integers in cents** (e.g., R$ 30,00 → `3000`)
- All timestamps are **UTC ISO 8601** strings

---

## 0. Authentication — Kiosk Token

Before any sync, the kiosk must authenticate and receive a JWT.

**Endpoint:** `POST /api/kiosk/v1/auth/token`
**Auth:** none (uses a one-time provisioning secret generated by the Core admin)

**Request:**
```json
{
  "kiosk_provisioning_secret": "one-time-secret-from-admin-panel"
}
```

**Response:**
```json
{
  "token": "eyJ...",
  "kiosk_id": "uuid",
  "tenant_id": "uuid",
  "expires_at": "2026-03-08T23:59:59Z"
}
```

The kiosk stores this token encrypted in SQLite (`app_config`). The token must be renewed on every successful sync (the pull rules response includes `jwt_expires_at`).

## 1. Pull — Rule Synchronization (Core → Satellite)

The kiosk downloads active tariffs and its own configuration to operate offline.

**Endpoint:** `GET /api/kiosk/v1/sync/rules`
**Auth:** `Authorization: Bearer <kiosk_jwt>`

**Response:**
```json
{
  "tenant_id": "uuid",
  "kiosk_id": "uuid",
  "kiosk_status": "active",
  "jwt_expires_at": "2026-03-08T23:59:59Z",
  "synced_at": "2026-03-01T12:00:00Z",
  "active_tariffs": [
    {
      "id": "uuid",
      "name": "Padrão Fim de Semana",
      "base_price": 3000,
      "base_minutes": 30,
      "additional_fraction_price": 100,
      "additional_fraction_minutes": 1,
      "tolerance_minutes": 5,
      "valid_from": "2026-01-01T00:00:00Z",
      "valid_until": null
    }
  ]
}
```

**Notes:**
- `kiosk_status: "blocked"` → kiosk must lock screen immediately (delinquent tenant or manual block)
- `valid_until: null` → tariff has no expiration
- Tariffs are versioned. After saving, old sessions must keep a reference to `tariff_id` for correct audit recalculation
- The kiosk must save the full tariff snapshot, not just the ID, because the tariff may change after sessions are stored

## 2. Push — Passive Session Synchronization (Satellite → Core)

The kiosk sends batched session data in the background when network connectivity is restored. Both active (ongoing) and completed sessions are sent.

**Endpoint:** `POST /api/kiosk/v1/sync/sessions`
**Auth:** `Authorization: Bearer <kiosk_jwt>`

**Request:**
```json
{
  "kiosk_id": "uuid",
  "sync_timestamp": "2026-03-01T12:00:00Z",
  "sessions": [
    {
      "local_id": "uuid",
      "child_name": "Maria",
      "guardian_name": "Ana Silva",
      "guardian_contact": "+5511999999999",
      "tariff_id": "uuid",
      "tariff_snapshot": {
        "base_price": 3000,
        "base_minutes": 30,
        "additional_fraction_price": 100,
        "additional_fraction_minutes": 1,
        "tolerance_minutes": 5
      },
      "check_in_at": "2026-03-01T10:00:00Z",
      "check_out_at": "2026-03-01T10:45:00Z",
      "calculated_total": 4500,
      "status": "completed",
      "notes": null
    }
  ]
}
```

**Session `status` values:**
| Value | Meaning |
|---|---|
| `active` | Child still playing (sync of current state) |
| `completed` | Normal check-out, payment collected |
| `canceled` | Session voided (operator correction, with log) |

**Response:**
```json
{
  "status": "success",
  "accepted_count": 1,
  "failed_sessions": [],
  "new_jwt": "eyJ...",
  "new_jwt_expires_at": "2026-03-08T23:59:59Z"
}
```

**`failed_sessions` item shape:**
```json
{
  "local_id": "uuid",
  "reason": "audit_mismatch",
  "server_calculated_total": 4600,
  "details": "Tariff version expired before check_out_at"
}
```

**Notes:**
- The kiosk must include `tariff_snapshot` because the Core needs to recalculate using the **tariff active at check-in time**, not the current one
- `guardian_contact` must be stored as a raw string (no format enforcement in Phase 1). Core may normalize/validate at ingestion
- On receiving a new JWT in the response, the kiosk must overwrite the stored token immediately
- Sessions with `failed_sessions` entries must NOT be removed from the local sync queue — they require human review

## 3. TypeScript Interface Definitions (Shared)

These live in `src/shared/types/sync.ts` and are used by both the Preload and Main process.

```typescript
export interface Tariff {
  id: string;
  name: string;
  base_price: number;         // cents
  base_minutes: number;
  additional_fraction_price: number;  // cents
  additional_fraction_minutes: number;
  tolerance_minutes: number;
  valid_from: string;         // UTC ISO 8601
  valid_until: string | null; // UTC ISO 8601
}

export interface SessionPayload {
  local_id: string;
  child_name: string;
  guardian_name: string;
  guardian_contact: string;
  tariff_id: string;
  tariff_snapshot: Omit<Tariff, 'id' | 'name' | 'valid_from' | 'valid_until'>;
  check_in_at: string;        // UTC ISO 8601
  check_out_at: string | null; // null if still active
  calculated_total: number;   // cents
  status: 'active' | 'completed' | 'canceled';
  notes: string | null;
}

export interface SyncRulesResponse {
  tenant_id: string;
  kiosk_id: string;
  kiosk_status: 'active' | 'blocked';
  jwt_expires_at: string;
  synced_at: string;
  active_tariffs: Tariff[];
}

export interface SyncSessionsResponse {
  status: 'success' | 'partial' | 'error';
  accepted_count: number;
  failed_sessions: FailedSession[];
  new_jwt: string | null;
  new_jwt_expires_at: string | null;
}

export interface FailedSession {
  local_id: string;
  reason: string;
  server_calculated_total?: number;
  details?: string;
}
```
